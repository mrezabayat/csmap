## In simple terms

Electron lets web developers build desktop apps using the same HTML, CSS, and JavaScript they use for websites. It bundles Chromium (the browser engine that powers Chrome) and Node.js into a single executable — so your app gets a full browser for rendering the UI and Node.js for accessing the file system, native OS features, and system resources. Write once in JavaScript, ship on Windows, macOS, and Linux. VS Code, Slack, Discord, WhatsApp Desktop, and GitHub Desktop are all Electron apps.

## More detail

**Architecture:**
- **Main process:** a Node.js process that manages the application lifecycle, native menus, system tray, file system access, and creates renderer windows. The "backend" of the app.
- **Renderer process:** a Chromium instance rendering HTML/CSS/JavaScript — one per window, like a browser tab. The "frontend."
- **IPC (Inter-Process Communication):** `ipcMain` / `ipcRenderer` — message passing between the Node.js main process and the Chromium renderer process. Renderer processes cannot access Node.js APIs directly (security isolation).
- **Context Bridge:** a safe API surface for the renderer to call limited main-process functions.

```javascript
// Main process — main.js
const { app, BrowserWindow, ipcMain } = require('electron');
ipcMain.handle('read-file', (event, path) => fs.readFileSync(path, 'utf8'));

// Renderer (preload script)
const { contextBridge, ipcRenderer } = require('electron');
contextBridge.exposeInMainWorld('api', {
  readFile: (path) => ipcRenderer.invoke('read-file', path)
});
```

**Why Electron works:**
- **Write once, run anywhere:** one codebase, one engineering team, three operating systems. Without Electron, you'd need separate Objective-C/Swift (macOS), C# WPF/WinForms (Windows), and GTK/Qt (Linux) teams.
- **Web technology ecosystem:** npm, React, Vue, and every web tool works in Electron.
- **Native access via Node.js:** file system, child processes, native modules — things not available in a browser.

**Why Electron is criticised:**

*Memory and disk footprint:* each Electron app bundles a full Chromium (~150 MB) and Node.js. A typical Electron app is 100–300 MB installed. If you have Slack, Discord, VS Code, and Spotify open simultaneously, you have 4 Chromium instances in memory — potentially 1–2 GB of RAM just for the browsers. VS Code uses ~500 MB RAM; a native equivalent would use ~50 MB.

*CPU usage:* Chromium's rendering pipeline, V8's JIT, and JavaScript execution consume more CPU than native frameworks. Battery drain is a common complaint.

*Startup time:* loading Chromium and Node.js from disk takes 1–3 seconds on average hardware. Native apps typically start in 100–500ms.

**Alternatives:**
- **Tauri (Rust):** uses the OS's native WebView (WebKit on macOS/Linux, EdgeHTML/WebView2 on Windows) instead of bundling Chromium. Apps are 5–10 MB installed, use much less RAM. Trade-off: less WebView consistency across platforms; Rust backend instead of Node.js.
- **Flutter Desktop:** Google's UI framework using its own rendering engine (Skia/Impeller). True native feel, good performance, Dart language.
- **Qt / wxWidgets / .NET MAUI:** truly native frameworks.

## Why it matters

Electron's model — web UI + native process — resolved a decades-long problem of cross-platform desktop development. VS Code's success (50+ million users) proved that an Electron app could be a best-in-class development tool. Understanding Electron clarifies the architecture of the majority of popular desktop apps you use, the trade-offs of web-first development tools, and why RAM usage is higher than expected. As Tauri and lighter alternatives mature, understanding the trade-offs helps make informed platform choices.

## Real-world examples

- VS Code (Microsoft): the most popular code editor worldwide, written in TypeScript on Electron.
- Slack: desktop Slack is an Electron app.
- Discord: Electron on desktop.
- GitHub Desktop, Figma (now native), Notion, 1Password 7, Twitch.
- WhatsApp Desktop: Electron (Windows), native on macOS.

## Common misconceptions

- **"Electron apps are web apps pretending to be desktop apps."** Electron apps have full access to the file system, native menus, system tray, notifications, and OS APIs — capabilities unavailable in a browser. They are genuinely desktop applications.
- **"Electron is always slow."** VS Code's startup, editing performance, and extension system are excellent despite being Electron. Performance depends on the app's implementation; Electron doesn't preclude performant apps.

## Learn next

Electron's model is being challenged by [progressive web apps](/t/progressive-web-app) (browser-based with offline and install capabilities) and lighter frameworks like Tauri. [WebAssembly](/t/webassembly) can run inside Electron's Chromium for compute-intensive logic. Understanding [Node.js](/t/nodejs) is essential for Electron's main process.
