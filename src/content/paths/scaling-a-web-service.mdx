Most "scalable web app" advice is folk wisdom. This path is the underlying mental model: where the bottlenecks actually appear as load grows, what each scaling tool buys you, and what it costs.

Start with single-machine speed (indexes, query plans, caches). Move to multi-machine (replication, sharding, CDN). End with the system architecture decisions (CAP, eventual consistency) that distinguish "works at scale" from "fails subtly in production".
